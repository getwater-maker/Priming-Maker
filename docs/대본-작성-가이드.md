# Priming 대본 작성 가이드 (롱폼 16:9 · 쇼츠 9:16 통합)

> 이 문서는 **Priming 영상 제작 앱이 자동으로 읽어 들이는 대본(.md) 형식**을 정의합니다.
> 이 가이드를 받은 AI는 아래 규칙을 **그대로** 지켜 대본을 작성하세요.
> 사용자는 결과 `.md` 파일을 Priming 앱에 첨부해 TTS·이미지·영상(I2V)·자막을 자동 생성합니다.
> 형식을 벗어나면 장면 분리·자막·이미지/영상 매핑이 깨지므로 규칙을 임의로 바꾸지 마세요.

## 0. AI에게 주는 작업 지시 (먼저 읽기)

- 출력은 **순수 마크다운(.md) 텍스트 하나**. 전체를 코드펜스로 감싸지 말 것.
- **롱폼인지 쇼츠인지 먼저 정한다.** 둘은 형식이 다르다 (아래 Part A / Part B).
- **이미지 프롬프트와 비디오(I2V) 프롬프트를 함께 작성한다.**
  - 이미지 프롬프트: **모든 장면(그룹)** 에 1개씩.
  - 비디오(I2V) 프롬프트: **움직임이 필요한 장면**(롱폼=도입부, 쇼츠=훅·절정)에 작성.
  - 이미지/비디오 프롬프트는 **영문**으로 (생성 품질).
- 따옴표·이모지·특수기호는 음성 합성에서 제거되니 **본문 의미를 거기에 의존하지 말 것**.

## 공통 핵심 원칙

1. **문장 1개 = 자막·음성 클립 1개.** 마침표(`.` `!` `?`)로 끝나는 문장 단위로 잘린다.
2. **그룹(장면) 1개 = 이미지 1장 (+ 선택적으로 비디오 1개).**
3. 음성에서 제외되는 것:
   - `<!-- 메모 -->` (여러 줄 가능)

   - 이모지·기호 `★ ※ → ▶ 😀` 등

# Part A. 롱폼 (16:9, 긴 영상 1편)

## A-1. 핵심 규칙

- **헤딩(`##`/`###`) 1개 = 장면(그룹) 1개 = 이미지 1장.** 기본 분할이 **H3 모드**라 화면이 바뀔 지점마다 헤딩으로 끊는다.
- ⚠️ **헤딩이 하나도 없으면 대본 전체가 1장면(이미지 1장)** 이 된다. 장면 수 = 헤딩 수.
- **도입부**: H1/H2 헤딩에 **"도입"** 글자가 들어가면 그 아래가 도입부(영상 우선 생성 구간).

## A-1.5. 실전 2단 구조 (현장 검증 — 파서가 그대로 받아준다)

> A-1의 "헤딩 1개 = 장면 1개"를 실전에서 더 정교하게 쓰는 검증된 방식. 파서 코드로 동작을 확인했다.

- 실전에선 **`## 섹션 소제목`(H2) + 그 아래 여러 개의 `### 장면 그룹`(H3)** 2단으로 쓴다. H2 = 글 흐름의 챕터, H3 = **이미지 1장 단위(장면)**.
- **H2 소제목에는 프롬프트를 달지 않는다**(바로 H3로 내려간다). 파서는 **본문 문장이 없는 헤딩(빈 H2)은 그룹으로 만들지 않고 무시**하므로 "빈 장면"이 생기지 않는다.
- 이미지/비디오 프롬프트는 **H3마다** 단다. (장면 수 = H3 수)
- **`## 도입부`(H2)** 아래의 H3들은 도입부 플래그를 **상속**한다 → 도입부를 여러 H3로 쪼개도 전부 도입부(영상 우선)로 인식된다.
- **파서 관용 (코드 검증됨 — 안심하고 써도 된다):**
  - `> 🖼️ 이미지:` 의 **이모지는 무관** — 키워드 `이미지:` 만 매칭한다.
  - `> 🎬 비디오(I2V):` 처럼 **"비디오" 뒤 `(I2V)` 접미사 OK** (정규식 `비디오[^:]*:`). `영상:` 도 인식.
  - **이미지/비디오가 아닌 `>` 줄**(작업 주석 `> 🎯`, `> 〔메모〕`, `> 🎨 …` 등)은 **파서가 자동으로 낭독·자막에서 제외**한다. 발행 전 굳이 지우지 않아도 된다.

## A-2. 프롬프트 문법 (롱폼) — `>` 인용줄, 백틱 없음

```markdown
## 1. 겹겹의 담장
> 🖼️ 이미지: a grand Joseon palace with five layered gates at dawn, cinematic wide shot
> 🎬 비디오: slow push-in toward the main gate, subtle drifting fog
경복궁에 들어서려면 다섯 개의 문을 지나야 했습니다.
각 문에는 무장한 군사들이 배치되어 있었습니다.
```

- `> ... 이미지: <영문 프롬프트>` → 그 섹션의 **이미지 프롬프트**
- `> ... 비디오: <영문 프롬프트>` (또는 `영상:`) → **비디오(I2V) 프롬프트**
- 이모지(🖼️/🎬)는 있어도 없어도 됨. 핵심은 **`이미지:` / `비디오:` 키워드 + 콜론**.
- **백틱(\`) 쓰지 말 것** — 롱폼은 콜론 뒤 전체를 프롬프트로 읽는다.

## A-4. 이미지/비디오 프롬프트 작성 규칙 (영상 품질의 핵심)

> 프로그램은 대본 프롬프트를 **그대로** 이미지 엔진에 넣는다 — 최종 = `<스타일>, <대본 프롬프트>, no text, no watermark`. 화풍·비율은 프로그램이 자동으로 입히므로 **대본엔 넣지 않는다.** (본문 정지 이미지엔 프로그램이 Ken Burns 줌/팬을, 도입부엔 I2V 영상을 자동 적용한다.)

**이미지 프롬프트 5요소** — 이미지는 그룹마다 **독립 생성**이라 앞 그룹의 시대·인물을 기억 못 한다. 그래서 **매 프롬프트가 혼자 완결**돼야 한다:
1. **시대·지역** — 예외 없이 명시 (예: `Joseon-dynasty Korea, Korean` / `16th-century Europe`). ⛔ "당연히 아니까" 생략 금지(= 엉뚱한 시대·지역으로 튀는 원인).
2. **인물** — 성별·연령·의상. 주인공은 **매 프롬프트에서 같은 인물 카드를 재진술**(컷 사이 룩 흔들림 방지).
3. **상황** — 그 그룹 본문의 장면.
4. **톤·분위기** — 조명·색·계절·감정.
5. (인물 없는 컷) **상징·소품**으로.

**금지·필수:**
- ⛔ **화풍·비율 토큰 금지** (`cinematic`·`16:9`·`aspect ratio`·스타일 단어) — 프로그램 스타일과 충돌한다. 끝엔 `no text, no watermark` 만.
- **켄번스 여백** — 본문 이미지는 줌/팬할 **공간을 남긴다**(꽉 찬 클로즈업 금지). 텍스트 오버레이 자리도 비워둔다.
- **도입부 비디오(I2V)** = **모션 묘사**(`slow push-in` 등). 본문은 이미지만 쓴다(줌/팬은 프로그램이 자동).
- **🛡 안전필터 회피** — 이미지 엔진(Gemini류)은 폭력·유혈·무기·시신·고통을 'unsafe'로 차단한다. 직접어 대신 **그림자·여파·상징·표정·조명**으로 표현한다. (예: `kill·blood·sword·execution·corpse` → 빈 형장·꺼진 등불·시든 꽃·뒷모습·갑옷 호위)
- 프롬프트는 **영문**으로.

> ※ A-4는 **형식·기술 규칙**이라 주제(역사·인물·과학 등)와 무관하게 **모든 프로젝트에 그대로 적용**된다. 주제별 "내용·톤·페르소나"는 각 프로젝트의 대본 지침이 따로 정한다. **이 가이드를 다른 프로젝트에 복사해 넣고, 그 프로젝트의 내용 지침과 함께 쓰면 Priming 100% 호환 대본이 나온다.**

# Part B. 쇼츠 (9:16, 한 파일에 3편)

## B-1. 핵심 규칙

- 한 `.md` 파일에 **`## 쇼츠 1` / `## 쇼츠 2` / `## 쇼츠 3`** → 쇼츠 3편이 한 번에 생성된다.
- 각 쇼츠는 **`**🎬 그룹 N ｜단계 (모드)**`** 헤더로 장면을 나눈다. **그룹 1개 = 이미지 1장**.
- 구분자 **`｜`(전각 세로줄)** 를 꼭 쓴다: `**🎬 그룹 1 ｜훅 (이미지 → 비디오)**`.
- 각 그룹 문장은 `- 음성/자막:` 아래에 **`  - 문장`** 으로 나열.
- **훅 자막(첫 프레임 제목)**: `- 훅 자막(첫 프레임): 윗줄 / 아랫줄` — 슬래시(`/`)로 2줄 분할.

## B-2. 프롬프트 문법 (쇼츠) — `-` 리스트, 백틱 필수

```markdown
**🎬 그룹 1 ｜훅 (이미지 → 비디오)**
- 음성/자막:
  - 첫 문장입니다.
  - 둘째 문장입니다.
- 🖼️ 이미지: `english image prompt, vertical 9:16, cinematic`
- 🎬 → 비디오(I2V): `slow push-in on her face, gentle falling snow`

**🎞️ 그룹 2 ｜본론 (이미지 + 모션)**
- 음성/자막:
  - 본문 문장입니다.
- 🖼️ 이미지: `english image prompt`
- 🎞️ 모션: slow zoom-in, 좌→우 팬
```

- `- 🖼️ 이미지: \`...\`` → 이미지 프롬프트 (**백틱 안에**)
- `- 🎬 → 비디오(I2V): \`...\`` → 비디오(I2V) 프롬프트 (**백틱 안에**, 줄이 `- 🎬` 로 시작)
- `- 🎞️ 모션: ...` → 켄번스 힌트(영상 아님, 백틱 없이). 비디오를 안 만드는 그룹용.
- 권장: **훅·절정 그룹 = 🎬 비디오 프롬프트**, 나머지 = 🎞️ 모션.

- 🖼️ 이미지: `(english image prompt)`

**🎞️ 그룹 2 ｜본론 (이미지 + 모션)**
